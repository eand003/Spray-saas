1. Criação e Gestão de Usuários e Perfis

O Spray Precision Manager utiliza o Supabase Auth para gerenciar credenciais e login com criptografia de ponta. A criação de novos usuários da equipe comercial ou de campo é facilitada diretamente pelo próprio aplicativo.

Como criar novas contas para a equipe:

  1. Abra o aplicativo em seu navegador ou celular.
  2. Na tela de login, clique no link "Cadastre-se grátis" na parte inferior.
  3. Insira os dados do colaborador: Nome Completo, E-mail Corporativo, Senha (mínimo de 6 dígitos) e WhatsApp de contato.
  4. No seletor de Função / Perfil de Acesso, selecione a atribuição operacional correspondente:
Perfil Identificador Interno Principais Permissões
Administrador Geral admin Acesso financeiro total, fluxo de caixa corporativo, aprovação e liberação de comissões, cadastro de despesas e override de preço mínimo de bico.
Vendedor / Parceiro seller Acesso ao CRM (somente seus próprios leads e clientes), visitas técnicas, simulador agrícola de parcelamento e visão dos seus próprios ganhos.
Técnico / Instalador technician Registro de visitas operacionais e laudos de instalação em pulverizadores dos clientes.
💡

Automação de Perfil por Trigger: Você não precisa registrar o usuário em tabelas separadas. O banco de dados possui uma trigger segura chamada sp_on_auth_user_created que escuta as novas contas criadas no Auth do Supabase e gera instantaneamente a linha correspondente de perfil na tabela sp_profiles.

2. Gestão Financeira e Acesso Administrador

O perfil de Administrador Geral (admin) é o único habilitado a ver o módulo contábil e de faturamento global da empresa. Quando você acessa a aba Financeiro com esse perfil, o sistema habilita cinco painéis operacionais.

A. Fluxo de Caixa

Apresenta a saúde financeira consolidada da Spray Precision. O painel compila dinamicamente os valores de todas as tabelas contábeis:

  • Saldo Projetado: (Total Recebido + Contas a Receber Vencer/Vencidos) - (Despesas Pagas + Contas a Pagar).
  • Ajuda a antecipar furos de caixa ou programar investimentos em estoque e peças baseado nos prazos agrícolas vigentes.

B. Módulo de Vendas

Aqui você acompanha o faturamento comercial e gerencia as vendas dos kits eletrostáticos da equipe:

  • Módulos Isoladores vs Bicos Físicos: No cadastro de venda, você discrimina o total de bicos físicos da barra e o espaçamento (35cm ou 50cm). A engenharia determina quantos módulos isoladores (bicos eletrostáticos) realmente compõem o kit comercializado (isolador duplo reduz a quantidade necessária em espaçamentos de 35cm). O sistema calcula o valor final baseado no total de módulos de fato cobrados.
  • Override de Preço Mínimo: A engenharia de campo estipula o custo unitário mínimo de tabela em R$ 1.100,00 por módulo. Vendedores comuns são terminantemente bloqueados caso tentem vender abaixo dessa tabela. O administrador recebe um aviso na tela permitindo autorizar (override) um desconto estratégico se necessário.
  • Edição e Segurança: O painel permite alterar ou excluir vendas incorretas. Contudo, por questões de compliance contábil, se o cliente já pagou qualquer parcela daquela venda, o sistema bloqueia a edição/exclusão de forma preventiva.

C. Contas a Receber

Gerencie o faturamento das vendas parceladas nos prazos de safra ou safra/safrinha dos agricultores:

  • Confirmação de Recebimento (Baixa): Quando o cliente efetua o pagamento de uma parcela (via Pix ou Boleto), o administrador clica no botão de baixa.
  • Liberação de comissão integrada: No momento exato em que a baixa da parcela é dada no sistema, o banco de dados localiza a comissão proporcional do vendedor correspondente àquela parcela e a altera automaticamente de **Prevista (Expected)** para **Liberada (Released)** no extrato do vendedor.

D. Contas a Pagar (Despesas)

Aba utilizada para lançar custos da empresa. Ela centraliza a saída operacional:

  • Permite controle de despesas corporativas como: Logística de Campo, Compra de Matéria-prima (mangueiras, conectores, isoladores), Impostos, etc.
  • Assegura que as projeções de saldo líquido no fluxo de caixa permaneçam exatas.

E. Comissões Geral

Consolida as comissões devidas aos vendedores. Toda venda gera uma comissão dividida nas mesmas parcelas e vencimentos do contrato comercial.

As comissões passam pelos seguintes estados de faturamento:

  1. Prevista (Expected): Vinculada a parcelas do cliente que ainda estão em aberto.
  2. Liberada (Released): Disponível para repasse ao vendedor, pois a respectiva parcela já foi recebida pela empresa.
  3. Paga (Paid): Repasse já realizado. Ao marcar como paga, o sistema gera automaticamente um lançamento de saída idêntico no módulo Contas a Pagar na categoria de despesa "Comissão", registrando a saída contábil líquida sem retrabalho.

3. Supabase Avançado: Alterando Perfis Manualmente

Se você precisar promover ou alterar a função de um usuário já existente manualmente (por exemplo, promover um Vendedor para Administrador Geral), o painel do Supabase pode restringir a edição visual direta ou o comando SQL UPDATE direto na tabela auth.users devido a políticas de segurança e permissões (ERROR: 42501: permission denied for table users).

A forma mais rápida, segura e 100% garantida de fazer isso é utilizando uma função auxiliar com privilégios de superusuário (SECURITY DEFINER).

Passo a passo para promover/alterar perfil via SQL Editor:

  1. Acesse o seu console em supabase.com e entre no projeto correspondente.
  2. Vá no menu lateral e acesse o SQL Editor.
  3. Crie a função de segurança executando o script abaixo (você só precisa criar esta função uma vez no projeto): 
    CREATE OR REPLACE FUNCTION public.sp_promote_user(user_email TEXT, new_role TEXT)
RETURNS VOID SECURITY DEFINER SET search_path = public, auth AS $$
BEGIN
  -- 1. Atualiza os metadados internos de autenticação do Supabase
  UPDATE auth.users
  SET raw_user_meta_data = COALESCE(raw_user_meta_data, '{}'::jsonb) || jsonb_build_object('role', new_role)
  WHERE email = user_email;

  -- 2. Atualiza a tabela sp_profiles do SaaS
  UPDATE public.sp_profiles
  SET role = new_role
  WHERE email = user_email;
END;
$$ LANGUAGE plpgsql;
  4. Com a função criada, execute o comando abaixo para promover o usuário desejado (ajuste o e-mail e a role para 'admin', 'seller' ou 'technician'): 
    SELECT public.sp_promote_user('admin@spray.com', 'admin');
  5. Clique no botão verde Run.

Pronto! As duas tabelas (auth.users e sp_profiles) serão atualizadas simultaneamente e com segurança. O usuário terá acesso instantâneo ao novo painel no próximo login ou simplesmente atualizando a página.
